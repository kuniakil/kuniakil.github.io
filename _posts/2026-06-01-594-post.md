在將多個服務遷移至 Kubernetes 後，我們建立了一套更強大的「神經系統」——**Jelly-Lab 數位大動脈**。這套系統能將 AI 產出的技術文件，自動、同步地發布到個人網站與私有知識庫中。

## 1. 輕量化部署與架構優化

為了在 8GB RAM 的 Mac 上穩定運行，我們對 n8n 進行了深度瘦身：
*   **引擎**：採用 SQLite 模式，省去了額外的資料庫容器開銷。
*   **儲存**：透過 `hostPath` 將工作流資料保存在 `/Users/mlee/n8n-data`。
*   **對外門戶**：利用 Cloudflare Tunnel 實現 `n8n.3pm.lol` 的安全穿透。

## 2. 雙重分支自動化管線 (v2.0)

這是一條具備「並行處理」能力的生產線，能在一次觸發中完成多重任務：

### 樞紐：Webhook (接收意圖)
*   統一接收來自 AI Agent 的 JSON 封包（包含標題與原始內容）。

### 支線 A：個人網站發布 (Jekyll 規範)
*   **加工**：自動生成 `YYYY-MM-DD-slug.md` 檔名，並注入 Jekyll 所需的 Front Matter。
*   **發布**：直接推送到 `kuniakil.github.io` 倉庫的 `_posts/` 目錄。

### 支線 B：LLM-wiki 知識整理 (Karpathy SOP)
*   **邏輯**：利用 JavaScript (Code Node) 計算內容的 **SHA256**、產生語義化的 **Slug**。
*   **雙重寫入**：
    1.  存入 `raw/articles/` 作為原始檔案。
    2.  存入 `concepts/` 作為結構化 Wiki 頁面，並自動標註 **Provenance (溯源)**。

## 3. 實戰與未來：AI 驅動維運 (AIOps)

這套系統最迷人之處在於其「無感化」。身為人類，我們不再需要去打 `curl` 或是手動複製檔案。這一切都由 AI Agent 與 n8n 配合完成：
*   **AI** 負責思考與內容生成。
*   **n8n** 負責執行瑣碎的搬運與格式化工作。

這正是 **AIOps (AI 維運)** 的具體實踐：讓基礎設施具備理解與執行的能力。
