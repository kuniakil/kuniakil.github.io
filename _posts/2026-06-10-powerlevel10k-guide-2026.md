# 🎨 Powerlevel10k 終端機主題配置指南

`Powerlevel10k`（簡稱 p10k）是目前 Zsh 最受歡迎、速度極快且視覺效果最豐富的提示字元主題。它能將單調的命令列，轉化為能夠即時顯示 **Git 分支狀態**、**Kubernetes 當前叢集**、執行時間等關鍵資訊的儀表板。

---

## 💡 為什麼要使用 Powerlevel10k？
* **極速響應**：採用非同步與預編譯技術，即使在極大的 Git 專案（如 Linux Kernel）中敲 Enter，也完全不會有任何卡頓感。
* **情境導向（Contextual）**：只在需要時顯示資訊。例如，只有當您在 Git 目錄下才會顯示 Git 資訊；只有在執行 K8s 指令或目錄下有 K8s 設定時，才會在右側顯示 K8s 狀態。
* **開箱即用**：提供極度親民的互動式設定精靈，不需要手動修改複雜的設定檔。

---

## 🌿 Git 狀態燈號與圖示說明

當您進入一個 Git 專案目錄（如 `~/kubernetes`）時，左下角會顯示當前分支的詳細資訊。以下是常見的狀態指示：

### 1. 顏色狀態
* 🟢 **綠色背景/文字**：代表當前分支非常乾淨（Clean），沒有任何修改。
* 🟡 **黃色背景/文字**：代表目前有檔案被修改、新增，但尚未 commit（Dirty）。

### 2. 常見符號對照
當分支處於 Dirty 狀態時，分支名稱後方會出現以下符號：

| 符號 | 代表意義 | 說明 |
| :---: | :--- | :--- |
| **`?`** | **未追蹤檔案（Untracked）** | 目錄中有新建立的檔案，但尚未執行 `git add`。 |
| **`!`** | **已修改但未暫存（Modified）** | 已被 Git 追蹤的檔案被修改了，但尚未 `git add`。 |
| **`+`** | **已暫存（Staged）** | 檔案已準備好，等待被 commit（已執行過 `git add`）。 |
| **`⇡N`** | **領先遠端（Ahead）** | 您本地有 `N` 個 commit 尚未 `git push` 到 GitHub。 |
| **`⇣N`** | **落後遠端（Behind）** | GitHub 遠端有 `N` 個新的 commit，您需要執行 `git pull`。 |

* **實際效果範例**：
  `feature/caddy-ingress *⇡1` 代表您目前在 `feature/caddy-ingress` 分支，有未提交的修改（`*`），且本地領先遠端 1 個 commit（`⇡1`）。

## ☸ Kubernetes (K8s) 狀態顯示

Powerlevel10k 最受 K8s 工程師喜愛的功能，就是會在畫面**右下角**即時顯示當前的 Kubernetes 狀態。

* **顯示樣式**：通常會顯示一個舵輪圖案與叢集名稱，例如 `☸ orbstack`。
* **智慧顯示機制**：為了保持畫面簡潔，K8s 資訊**不會**一直亮著。只有在以下情況下才會顯示：
  1. 您剛剛敲了 `kubectl`、`helm`、`k9s` 等 K8s 相關指令。
  2. 您當前目錄下包含 Kubernetes 設定檔或範本檔案。
* **好處**：防止「非對稱操作災難」！在多叢集環境中，這能確保您隨時知道自己是在本地開發叢集（OrbStack），還是在生產環境叢集下發送指令。

## 🛠️ 設定與自訂

### 1. 重新配置樣式
如果您對當前的樣式不滿意，或是字型圖案跑掉，隨時可以在終端機輸入以下指令重新啟動設定精靈：
```bash
p10k configure
```
這會重新帶您走一遍字型測試與排版風格挑選。

### 2. 進階自訂設定檔
您的所有個人設定都儲存在家目錄下的：
* 📁 `~/.p10k.zsh`

這是一個標準的 Zsh 腳本，裡面有極為詳細的註解。您可以透過修改此檔案來調整要顯示哪些模組（如：CPU 溫度、電池電量、AWS 帳號、Python 虛擬環境等）。
